# Contributing

Keep changes small, specific, and easy to review.

## Setup

```bash
nvm use
npm install
npm run dev
```

Before opening a pull request, run:

```bash
npm run build
```

## Working Rules

- Follow [COPY_RULES.md](./COPY_RULES.md) for all site copy and README edits.
- Keep the product framing plain: this is a skill for AI agents used in tools like Codex, Claude Code, Cursor, and similar setups.
- Do not reformat unrelated files or mix tidy-ups into feature work.
- Keep generated or asset-heavy changes scoped to the task you are actually doing.
- If you change the public-facing copy, keep the README and landing page aligned.
